## Cargo is npm + tsc + jest in one

If you have used `npm` (or `yarn`/`pnpm`), Cargo will feel immediately familiar. It manages dependencies, builds your project, runs your code, and runs your tests — all under one roof.

## Creating a new project

```bash
# Node / TypeScript
mkdir my-app && cd my-app
npm init -y
npm install typescript --save-dev

# Rust
cargo new my-app   # creates a binary (has fn main)
cargo new my-lib --lib   # creates a library (no fn main)
cd my-app
```

`cargo new` creates:
```
my-app/
├── Cargo.toml      # ← like package.json
├── Cargo.lock      # ← like package-lock.json
└── src/
    └── main.rs     # ← entry point
```

## Adding dependencies

```bash
# npm
npm install serde_json        # runtime dep
npm install --save-dev vitest # dev dep

# Cargo
cargo add serde_json          # runtime dep
cargo add --dev pretty_assertions  # dev dep
```

`cargo add` updates `Cargo.toml` automatically, exactly like `npm install`.

## Core subcommands

```bash
cargo build          # compile (debug, fast, unoptimized)
cargo build --release  # compile (optimized, slower build)
cargo run            # compile + execute
cargo run -- --port 8080  # pass args to your binary
cargo check          # type-check only, no binary (very fast)
cargo test           # run all tests
cargo test my_fn     # run tests whose name contains "my_fn"
cargo doc --open     # generate + open HTML docs
cargo clean          # delete target/ directory
```

## Cargo.lock — commit it or not?

```
# Binary / application  →  COMMIT Cargo.lock (reproducible builds)
# Library crate         →  DO NOT commit Cargo.lock (let consumers pick versions)
```

This mirrors npm's convention: applications commit `package-lock.json`; published libraries typically do not.

<RsOnly title="edition = 2021">

Rust releases a new **edition** every three years (2015, 2018, 2021). An edition may introduce breaking syntax changes, but the compiler can migrate your code automatically with `cargo fix --edition`. Always use `edition = "2021"` for new projects — it enables let-else, the prelude improvements, and better closure captures.

</RsOnly>
