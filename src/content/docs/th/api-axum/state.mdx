## ปัญหา: การแชร์ข้อมูลข้ามหลาย handlers

Handlers ของคุณต้องการเข้าถึง shared resources — database pool, config, cache client ใน Express คุณแนบสิ่งต่าง ๆ กับ `app.locals` หรือส่งผ่าน closure ใน NestJS Dependency Injection จัดการให้อัตโนมัติ ใน Axum คุณใช้ `State` extractor

## Express `app.locals` vs Axum `State`

export const tsState = `// Express — attach to app.locals
import express from 'express';
import { Pool } from 'pg';

const app = express();
app.locals.db = new Pool({ connectionString: process.env.DATABASE_URL });

app.get('/users', async (req, res) => {
  const db: Pool = req.app.locals.db; // untyped, runtime cast
  const { rows } = await db.query('SELECT * FROM users');
  res.json(rows);
});`;

export const rsState = `use axum::{extract::State, routing::get, Json, Router};
use sqlx::PgPool;
use std::sync::Arc;

#[derive(Clone)]
struct AppState {
    db: PgPool,
}

async fn list_users(
    State(state): State<Arc<AppState>>,
) -> Json<Vec<User>> {
    let users = sqlx::query_as!(User, "SELECT * FROM users")
        .fetch_all(&state.db)
        .await
        .unwrap();
    Json(users)
}

#[tokio::main]
async fn main() {
    let db = PgPool::connect(&std::env::var("DATABASE_URL").unwrap())
        .await
        .unwrap();

    let state = Arc::new(AppState { db });

    let app = Router::new()
        .route("/users", get(list_users))
        .with_state(state);

    let listener = tokio::net::TcpListener::bind("0.0.0.0:3000").await.unwrap();
    axum::serve(listener, app).await.unwrap();
}`;

## NestJS providers vs Axum State

ใน NestJS services คือ singletons ที่สร้างโดย IoC container และ inject เข้าไปใน controllers ผ่าน constructor ใน Axum คุณสร้าง state หนึ่งครั้งใน `main`, wrap ด้วย `Arc` และแนบกับ `Router` ทุก handler ที่ต้องการมันประกาศ `State(state): State<Arc<AppState>>`

export const tsNestDI = `// NestJS
@Injectable()
export class UsersService {
  constructor(private readonly db: DatabaseService) {}

  async findAll() {
    return this.db.query('SELECT * FROM users');
  }
}

@Controller('users')
export class UsersController {
  constructor(private readonly usersService: UsersService) {}

  @Get()
  findAll() {
    return this.usersService.findAll();
  }
}`;

export const rsAxumState = `// Axum — no framework magic, just Arc + State
use axum::{extract::State, Json};
use std::sync::Arc;

#[derive(Clone)]
struct AppState {
    db: PgPool,
    config: AppConfig,
}

async fn list_users(
    State(state): State<Arc<AppState>>,
) -> Json<Vec<User>> {

}`;

## ทำไมต้องใช้ `Arc`?

`Arc<T>` คือ atomically reference-counted smart pointer Axum handlers ทำงานพร้อมกันบน Tokio tasks หลายตัว หากไม่มี `Arc` การส่ง `AppState` ไปยัง handlers หลายตัวจะต้องทำสำเนา ด้วย `Arc` handlers ทั้งหมดถือ reference ไปยัง allocation เดียวกัน — ไม่มีสำเนา ไม่มี locks สำหรับข้อมูล read-only

```rust
// AppState must implement Clone so Axum can clone the Arc per handler
#[derive(Clone)]
struct AppState {
    db: PgPool,           // PgPool is already Clone + Arc internally
    config: AppConfig,    // derive Clone on AppConfig
}
```

ถ้า state ของคุณมีข้อมูลที่ mutable (เช่น in-memory cache) ให้ wrap เฉพาะส่วนที่ mutable:

```rust
use std::sync::{Arc, RwLock};

#[derive(Clone)]
struct AppState {
    db: PgPool,
    cache: Arc<RwLock<HashMap<String, String>>>,
}
```

<RsOnly title="Arc vs Mutex vs RwLock">

- `Arc<T>` — shared ownership, immutable access ใช้สำหรับ state ที่ read-only เช่น config หรือ DB pool
- `Arc<Mutex<T>>` — shared ownership, exclusive (write) access ทีละ task เดียวถือ lock ใช้สำหรับการ write ที่เกิดขึ้นน้อย
- `Arc<RwLock<T>>` — shared ownership, concurrent reads **หรือ** one exclusive write ใช้เมื่อ reads บ่อยและ writes น้อย

สำหรับ Axum applications ส่วนใหญ่ database pool (`PgPool`, `MongoClient`) thread-safe และ `Clone` อยู่แล้ว คุณสามารถวางมันใน `AppState` โดยตรงโดยไม่ต้องมี wrapping เพิ่มเติม

</RsOnly>

> **รันบนเครื่องของคุณ** — `State` ต้องการ Axum router ที่กำลังทำงานพร้อม `.with_state(state)` ใช้ `cargo run` และเรียก endpoint ด้วย `curl`
