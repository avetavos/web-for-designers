## Cargo คือ npm + tsc + jest ในเครื่องเดียว

ถ้าคุณเคยใช้ `npm` (หรือ `yarn`/`pnpm`) Cargo จะรู้สึกคุ้นเคยทันที มันจัดการ dependencies, build project, รัน code และรัน test — ทั้งหมดอยู่ในที่เดียว

## สร้าง project ใหม่

```bash
# Node / TypeScript
mkdir my-app && cd my-app
npm init -y
npm install typescript --save-dev

# Rust
cargo new my-app   # สร้าง binary (มี fn main)
cargo new my-lib --lib   # สร้าง library (ไม่มี fn main)
cd my-app
```

`cargo new` สร้างโครงสร้าง:
```
my-app/
├── Cargo.toml      # ← เหมือน package.json
├── Cargo.lock      # ← เหมือน package-lock.json
└── src/
    └── main.rs     # ← entry point
```

## เพิ่ม dependencies

```bash
# npm
npm install serde_json        # runtime dep
npm install --save-dev vitest # dev dep

# Cargo
cargo add serde_json          # runtime dep
cargo add --dev pretty_assertions  # dev dep
```

`cargo add` อัปเดต `Cargo.toml` อัตโนมัติ เหมือนกับ `npm install` ทุกประการ

## Subcommand หลัก

```bash
cargo build          # compile (debug, เร็ว, ไม่ optimise)
cargo build --release  # compile (optimise, build ช้ากว่า)
cargo run            # compile + execute
cargo run -- --port 8080  # ส่ง args ให้ binary
cargo check          # type-check เท่านั้น ไม่สร้าง binary (เร็วมาก)
cargo test           # รัน test ทั้งหมด
cargo test my_fn     # รัน test ที่ชื่อมี "my_fn"
cargo doc --open     # สร้าง + เปิด HTML docs
cargo clean          # ลบ directory target/
```

## Cargo.lock — commit หรือไม่?

```
# Binary / application  →  COMMIT Cargo.lock (build ที่ reproducible)
# Library crate         →  ไม่ commit Cargo.lock (ให้ผู้ใช้เลือก version)
```

นี่สอดคล้องกับ convention ของ npm: application commit `package-lock.json` แต่ library ที่ publish มักไม่ commit

<RsOnly title="edition = 2021">

Rust ออก **edition** ใหม่ทุก 3 ปี (2015, 2018, 2021) edition อาจมี syntax change ที่ breaking แต่ compiler สามารถ migrate code ให้อัตโนมัติด้วย `cargo fix --edition` สำหรับ project ใหม่ให้ใช้ `edition = "2021"` เสมอ — รองรับ let-else, prelude ที่ปรับปรุงแล้ว และ closure capture ที่ดีขึ้น

</RsOnly>
