## ยินดีต้อนรับสู่ Rust ขั้นสูง

คุณได้เรียนรู้ระบบ ownership, borrowing, และ type พื้นฐานของ Rust แล้ว ถึงเวลาปลดล็อกเครื่องมือที่ทำให้ Rust ทั้งมีพลังในการแสดงออกและมีต้นทุนเป็นศูนย์: ระบบ type ที่มีประสิทธิภาพพอที่จะแทนที่ framework ทั้งหมดในขณะ runtime ด้วยการรับประกันในขณะ compile time

บทเรียนนี้นำแต่ละแนวคิดไปเชื่อมกับสิ่งที่คุณรู้อยู่แล้วจาก TypeScript เพื่อให้คุณสร้าง mental model ได้อย่างรวดเร็ว

## สิ่งที่คุณจะได้เรียน

| บทเรียน | แนวคิด Rust | คู่เทียบใน TypeScript |
|---|---|---|
| Generics & Trait Bounds | `fn foo<T: Display>()`, `where` clauses | Generic functions พร้อม constraints `<T extends ...>` |
| Trait Objects | `dyn Trait`, `Box<dyn Trait>` | Interfaces, polymorphism, runtime dispatch |
| Closures & Iterators | `Fn`/`FnMut`/`FnOnce`, `.map()/.filter()/.collect()` | Arrow functions, `Array.prototype.map/filter/reduce` |
| Error Handling (ลึก) | `?` operator, `thiserror`, `anyhow` | `try/catch`, typed error classes |
| Macros | `macro_rules!`, derive macros | ไม่มีคู่เทียบตรงๆ — แต่ใกล้เคียง TypeScript decorators |
| Smart Pointers | `Box<T>`, `Rc<T>`, `RefCell<T>` | JavaScript references ที่ถูกจัดการโดย GC |

## ทำไมถึงสำคัญสำหรับนักพัฒนา TypeScript

TypeScript มี generics, interfaces, และ union error types Rust นำแนวคิดทั้งสามไปอีกขั้น:

- **Generics** ถูก monomorphise — compiler สร้างสำเนาเฉพาะสำหรับแต่ละ concrete type ไม่มี overhead จาก virtual dispatch
- **Trait objects** (`dyn Trait`) ให้ runtime polymorphism ที่คุณคุ้นเคยจาก TypeScript interfaces แต่คุณต้องเลือกใช้อย่างชัดเจน
- **Closures** จับค่าโดย reference หรือโดย move และ `Fn*` trait hierarchy บอก compiler ว่า closure ทำอะไรได้บ้าง
- **Error handling** ด้วย `?` และ `thiserror` ใช้งานง่ายพอๆ กับ `async/await` — และ type บังคับให้คุณจัดการทุก failure case
- **Macros** มีความ hygienic และทำงานบน AST — มีพลังมากกว่าและปลอดภัยกว่า TypeScript string-template code generation
- **Smart pointers** ให้การควบคุม heap allocation และ shared ownership อย่างละเอียดโดยไม่ต้องมี GC
