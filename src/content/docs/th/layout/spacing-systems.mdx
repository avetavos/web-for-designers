คุณรู้อยู่แล้วว่า spacing แบบสุ่มรู้สึกผิด เมื่อปุ่มมี padding `14px` ในที่หนึ่งและ `11px` ในอีกที่หนึ่ง ผู้ใช้รู้สึกได้ — แม้ว่าพวกเขาจะไม่สามารถบอกได้ว่าทำไม นักออกแบบใช้ spacing tokens เพื่อป้องกันสิ่งนี้ ใน CSS เราทำสิ่งเดียวกันพอดี: ผูกมัดกับ **spacing scale** และใช้เฉพาะค่าจาก scale นั้น

## ระบบ spacing 4/8px

spacing scale ที่ใช้กันมากที่สุดนั้นอิงตามตัวคูณของ 4 (บางครั้งเรียกว่า "4-point grid") หรือตัวคูณของ 8 ("8-point grid") ทุกค่า spacing ในโปรเจกต์ของคุณมาจากลำดับนี้:

| Step | 4pt scale | 8pt scale |
|---|---|---|
| 1 | 4px | 8px |
| 2 | 8px | 16px |
| 3 | 12px | 24px |
| 4 | 16px | 32px |
| 5 | 20px | 40px |
| 6 | 24px | 48px |

ใน CSS มักแสดงเป็นหน่วย `rem` (โดยที่ `1rem = 16px` ตามค่าเริ่มต้น):

| rem | เทียบเท่า px |
|---|---|
| `0.25rem` | 4px |
| `0.5rem` | 8px |
| `0.75rem` | 12px |
| `1rem` | 16px |
| `1.5rem` | 24px |
| `2rem` | 32px |

## สาม spacing properties

**`padding`** — พื้นที่ *ภายใน* element ระหว่าง border กับเนื้อหา ทำให้ element เองใหญ่ขึ้น คิดว่าเหมือน "horizontal/vertical padding" ใน Figma auto-layout frame

**`margin`** — พื้นที่ *ภายนอก* element ดัน neighbor ออกไป ไม่ทำให้ element ใหญ่ขึ้น — มันสร้างบัฟเฟอร์โซนรอบๆ

**`gap`** — พื้นที่ *ระหว่าง* children ใน Flex หรือ Grid container วิธีที่สะอาดและคาดเดาได้มากที่สุดในการเว้นระยะไอเทม — ไม่เพิ่มพื้นที่รอบขอบ เพิ่มเฉพาะระหว่างไอเทม

## เมื่อไรใช้อะไร

| สถานการณ์ | ใช้ |
|---|---|
| พื้นที่ระหว่างไอเทมใน flex หรือ grid container | `gap` |
| padding ภายในของปุ่ม, card หรือกล่อง | `padding` |
| ดัน element หนึ่งออกจาก sibling | `margin` |
| จัดกึ่งกลาง block element ในแนวนอน | `margin: 0 auto` |
| เว้นเนื้อหาห่างจากขอบหน้า | `padding` บน body/container |

    </div>
  </div>

    </div>
  </div>
</div>`;

export const spacingCss = `body { font-family: system-ui, sans-serif; }
.page {
  display: flex;
  flex-direction: column;
  gap: 1rem;
  padding: 1.5rem;

  border-radius: 8px;

สิ่งที่ลองได้:
- เปลี่ยน `padding` ของ `.card` จาก `1.5rem` เป็น `0.5rem` — สังเกตว่ามันรู้สึกแออัดแค่ไหน
- เปลี่ยน `gap` ของ `.page` จาก `1rem` เป็น `3rem` — สังเกตว่ามันโปร่งขึ้นแค่ไหน
- เปลี่ยน `gap` ของ `.actions` จาก `0.5rem` เป็น `2rem` — ปุ่มจะห่างออกจากกัน
- ลอง `padding: 2rem` บน `.card` — มันขยายให้พอดี

<Tip title="CSS custom properties เป็น spacing tokens">
เมื่อคุณคุ้นเคยแล้ว ย้ายค่า spacing ของคุณไปไว้ใน CSS custom properties (variables) ที่ด้านบนของ stylesheet: `--space-1: 0.25rem;` ถึง `--space-8: 2rem;` แล้วใช้ `padding: var(--space-4)` ทุกที่ นี่ mirror design tokens ได้พอดีและทำให้การเปลี่ยน spacing ทั่วโลกเป็นการอัปเดตบรรทัดเดียว — เหมือนกับการเปลี่ยนค่า token ใน design tool ของคุณพอดี
</Tip>
